01_sass소개와 문법

SASS - Syntactically Awesome StyleSheets의 줄임말

SCSS - Sassy CSS의 줄임말

CSS pre-processor(전기처리기)로서 css반복을 줄여서 시간을 절약할 수 있도록 해주는 css언어의 확장 언어이다.

css를 좀 더 효율적으로 작성하고 유지보수하기 쉽게 만들어 준다.

장점 - 변수 및 조건문 등의 새로운 문법을 통해 css를 구조화하여 작업할 수 있다.

단점 - 코드가 지저분해질 수 있고, 코드가 많아질수록 유지보수가 어렵다.

프로그램 설치는 sass공식사이트에서 gui, cli방식의 프로그램들 중에서 자신이 편한 프로그램을 다운 받아 설치한다.

vs code에서는 별도로 확장프로그램(live sass compiler)을 지원하기 때문에 왼쪽 메뉴 아이콘 'extensition'을 클릭하여 설치, 사용한다.

1. sass, scss문법 차이

SASS문법
중괄호({})와 세미콜론(;)이 필요없습니다.
들여쓰기(tab키)를 사용하여 코드 구조를 나타냅니다.
구문이 더 간결하고 Python스타일과 비슷합니다.
  sass작성 예시)
  $primary-color:#3498db;
  body
    background:$primaty-color;
    color:#fff;

    nav
    ul
      margin:0
      padding:0 
      list-style:none
    li
      display:inline-block
      margin-right:10px;
SCSS문법
CSS문법과 매우 유사합니다.
중괄호({})와 세미콜론(;)을 사용합니다.
기존 CSS와 호환성이 높습니다.
  sass작성 예시)
  $primary-color:#3498db;
  body{
    background:$primaty-color;
    color:#fff;
  }
    nav{
    ul{
      margin:0
      padding:0 
      list-style:none
      }
      li{
      display:inline-block
      margin-right:10px;
      } 
    }

둘중 어느 것을 작성하는 게 좋은가?

SCSS 권장:
CSS와 문법이 거의 같아 기존 CSS 파일과 통합하기 쉽습니다.
팀원들이 CSS에 익숙하다면 SCSS가 더 쉽게 받아들여질 수 있습니다.
최신 문법과 기능이 주로 SCSS에서 지원되며, SASS는 유지 보수용으로만 사용됩니다.
SASS 공식 문서에서도 SCSS 사용을 권장하고 있습니다.
SASS 사용이유 :
더 간결한 문법을 선호하거나, 기존 SASS 스타일을 유지해야 하는 프로젝트에 유리합니다.
결론
SCSS를 사용하는 것을 권장
더 널리 사용되며, 유지보수 및 협업에 유리합니다.
최신 기능 지원 및 CSS와의 호환성도 뛰어납니다.
